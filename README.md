# 🍪 쿠키박스 · 오븐룸 예약 시스템

쿠키박스 연습실 **오븐룸**의 예약/관리 페이지입니다.
**직접 운영하는 서버나 DB 없이** 구글 서비스만으로 동작합니다.

## 구성 (아키텍처)

```
 [고객/관리자 브라우저]
   frontend/ (정적 HTML·CSS·JS)   ← GitHub Pages 등 무료 호스팅
          │  fetch (JSON)
          ▼
 [Google Apps Script 웹앱]  ← "무료 서버리스" 백엔드 (구글 서버에서 실행)
          │
   ┌──────┴───────┐
   ▼              ▼
[구글 캘린더]   [구글 시트]
 예약 일정/       예약자 기록·
 가용성 관리       정산 데이터
```

- **왜 Apps Script?** 브라우저에서 캘린더/시트에 직접 쓰면 인증 키가 노출되고 고객마다 구글 로그인이 필요합니다. Apps Script 웹앱이 구글 인프라 위에서 그 역할을 대신해, 비밀값 노출 없이 "서버 없는" 백엔드가 됩니다.

## 폴더 구조
```
frontend/
  index.html        고객 예약 페이지
  admin.html        관리자 페이지 (토큰 필요)
  css/style.css
  js/config.js      ← 운영 설정 (공개됨: 비밀값 금지)
  js/api.js         Apps Script 호출 래퍼
  js/booking.js     고객 예약 로직
  js/admin.js       관리자 로직
apps-script/
  Code.gs           백엔드 (구글 서버 실행, 비노출)
  appsscript.json   권한/타임존 설정
docs/
  SETUP.md          구글 캘린더/시트/배포 단계별 가이드
```

## 설치
👉 **[docs/SETUP.md](docs/SETUP.md)** 를 순서대로 따라 하세요.

## 보안 모델 (요약)
- **공개(로그인 불필요)**: 빈 시간 조회, 예약 생성 → **개인정보를 반환하지 않음**
- **관리자 전용(토큰 필요)**: 예약자 이름·연락처 조회, 확정/취소
- 관리자 토큰·캘린더ID·시트ID 등 **비밀값은 프론트엔드 코드에 없고** Apps Script "스크립트 속성"(구글 서버)에만 저장 → 개발자도구로도 볼 수 없음
- **이중예약 방지**: 최종 판정은 서버(Apps Script)가 캘린더를 확인해 수행

## 현재 가정 (바꾸려면 알려주세요)
- 연동: 구글 캘린더 + 시트 **둘 다**
- 화면: 고객 예약 + 관리자 관리 **둘 다**
- 공간: `config.js` 의 `ROOMS` 로 개수 조절 (기본 예시 2개)
- 운영시간 10:00~22:00, 1시간 단위, 최대 3시간 — 모두 `config.js` 에서 변경
