# 🍪 오븐룸 예약 시스템

연습실 **오븐룸**의 예약/관리 페이지입니다.
**직접 운영하는 서버나 DB 없이** 구글 서비스만으로 동작합니다.

## 구성 (아키텍처)

```
 [고객/관리자 브라우저]
   정적 HTML·CSS·JS   ← GitHub Pages 무료 호스팅
          │  fetch (JSON)
          ▼
 [Google Apps Script 웹앱]  ← "무료 서버리스" 백엔드 (구글 서버에서 실행)
          │
   ┌──────┴───────┐
   ▼              ▼
[구글 캘린더]   [구글 시트]
 예약 일정/       예약자 기록·
 가용성 관리       정산 데이터
```

- **왜 Apps Script?** 브라우저에서 캘린더/시트에 직접 쓰면 인증 키가 노출되고 고객마다 구글 로그인이 필요합니다. Apps Script 웹앱이 구글 인프라 위에서 그 역할을 대신해, 비밀값 노출 없이 "서버 없는" 백엔드가 됩니다.

## 폴더 구조
```
index.html          고객 예약 페이지
admin.html          관리자 페이지 (아이디/비밀번호 로그인)
privacy.html        개인정보 처리방침
css/style.css
js/config.js        ← 운영 설정 (공개됨: 비밀값 금지)
js/api.js           Apps Script 호출 래퍼
js/booking.js       고객 예약 로직
js/admin.js         관리자 로직
apps-script/
  Code.gs           백엔드 (구글 서버 실행, 비노출)
  appsscript.json   권한/타임존 설정
docs/
  SETUP.md          구글 캘린더/시트/배포 단계별 가이드
```

## 보안 모델 (요약)
- **공개(로그인 불필요)**: 빈 시간 조회(시간대만)·요금 조회·예약 신청·쿠금통 잔여 조회(이름+뒷4자리 → 잔여 숫자만) → **개인정보를 반환하지 않음**
- **관리자 전용(아이디/비밀번호)**: 예약자 정보 조회, 확정/취소 — 비밀번호는 시트에 **해시로만 저장**, 처리 내역은 처리자와 함께 기록
- 캘린더ID·시트ID·해시 비밀키 등 **비밀값은 프론트엔드 코드에 없고** Apps Script "스크립트 속성"(구글 서버)에만 저장 → 개발자도구로도 볼 수 없음
- **이중예약 방지**: 신청·확정 모두 서버(Apps Script)가 시트 기준으로 잠금 후 겹침 검사
- **개인정보**: 공개 캘린더엔 마스킹 이름+연락처 뒷4자리만 표시, 예약 기록은 5년 후 자동 파기 (`privacy.html`)

## 운영 설정 위치
- 운영시간·시간 단위 등: `js/config.js` (`OPEN_HOUR / CLOSE_HOUR / SLOT_MINUTES`)
- **요금**: 시트 `요금표` 탭에서 수정 (최대 5분 내 자동 반영, 재배포 불필요)
- 쿠금통 회원·관리자 계정: 시트 `쿠금통` / `관리자` 탭
