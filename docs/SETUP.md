# 오븐룸 예약 시스템 · 설치/배포 가이드

서버·DB 없이 **구글 캘린더 + 구글 시트 + Google Apps Script(무료 서버리스)** 로 동작합니다.
아래 순서대로 한 번만 설정하면 됩니다.

---

## 1. 구글 캘린더 만들기
1. [Google 캘린더](https://calendar.google.com) → 왼쪽 "다른 캘린더" 옆 **+** → **새 캘린더 만들기**
2. 이름: `오븐룸 예약` → 만들기
3. 만든 캘린더 **설정** → 아래로 스크롤 → **캘린더 통합** → **캘린더 ID** 복사
   (예: `xxxxxxxx@group.calendar.google.com`) — 예약 시 일정이 여기에 **자동 등록**됩니다.
4. **사이트의 '예약 현황'에 이 캘린더를 표시**하려면 (선택):
   - 같은 설정 화면의 **액세스 권한** → **"일반 공개 사용 설정"** 체크 → **"모든 일정 세부정보 보기"**
     (일정 제목은 `19-22/김달*(8627)` 형식 — 마스킹된 이름과 연락처 뒷 4자리만 노출되고 전체 개인정보는 노출되지 않습니다.)
   - **캘린더 통합** → **"삽입 코드"** 의 `src="..."` 안의 URL
     (`https://calendar.google.com/calendar/embed?src=...`) 복사 → 5단계에서 사용

## 2. 구글 시트 만들기
1. [Google 시트](https://sheets.google.com)에서 새 스프레드시트 생성 → 이름 `오븐룸 예약기록`
2. 주소창 URL에서 **시트 ID** 복사
   `https://docs.google.com/spreadsheets/d/`**`여기가_시트ID`**`/edit`
   (시트 탭/헤더는 코드가 자동으로 생성하니 비워 둬도 됩니다.)
3. **쿠금통(선불 시간권) 회원 관리**: 같은 스프레드시트에 `쿠금통` 탭이 자동 생성됩니다.
   (헤더: 이름 | 연락처 | 충전시간 | 사용시간 | 잔여시간)
   - 회원 등록/충전: **이름·연락처·충전시간**을 직접 입력합니다. 조회는 이름 + 연락처 뒷 4자리 일치로 동작합니다.
   - **사용시간**은 예약이 확정될 때 자동으로 늘어나고, 확정을 취소하면 그만큼 되돌아옵니다. 직접 수정하지 마세요.
   - **잔여시간** 열은 자동 수식(충전시간-사용시간)이라 입력할 필요 없습니다.
4. **요금 관리**: `요금표` 탭이 자동 생성됩니다(현재 요금으로 채워짐).
   - A~D열: 평일/주말 시간대별 시간당 요금 — 행을 추가/삭제해 시간 구간을 바꿔도 됩니다.
   - F~G열: 인원 추가 기준(명)·인원당 추가요금·단원/공연팀 할인율(%)
   - 숫자를 고치면 **최대 5분 안에** 예약 페이지에 자동 반영되고, 입금액도 이 표 기준으로 서버가 계산합니다.
     (코드 수정·재배포 불필요)

## 3. Apps Script 프로젝트 만들기
1. [script.google.com](https://script.google.com) → **새 프로젝트**
2. 기본 `Code.gs` 내용을 지우고 `apps-script/Code.gs` 내용을 **전부 붙여넣기**
3. 상단 톱니(프로젝트 설정) → **"appsscript.json 매니페스트 파일 표시"** 체크
   → 편집기에 생긴 `appsscript.json` 을 `apps-script/appsscript.json` 내용으로 교체

### 3-1. 스크립트 속성(비밀값) 등록  ← 중요
프로젝트 설정(톱니) → **스크립트 속성** → 아래 3개 추가:

| 속성 이름 | 값 |
|---|---|
| `SHEET_ID` | 2단계에서 복사한 시트 ID **(필수)** |
| `ADMIN_TOKEN` | 관리자 비밀번호(원하는 임의 문자열, 길고 복잡하게) **(필수)** |
| `CALENDAR_ID` | 1단계에서 복사한 캘린더 ID **(선택 — 넣으면 예약 시 캘린더 자동 등록)** |
| `ADMIN_TOKENS` | (예비) `이름=토큰, 이름2=토큰2` 형식 — 아래 '관리자' 탭 방식을 권장 **(선택)** |

> **관리자 계정 운영(권장: 시트 `관리자` 탭)**: 스프레드시트에 `관리자` 탭
> (**아이디 | 비밀번호 | 닉네임**)이 자동 생성됩니다.
> **아이디(+닉네임)만 적으면 끝** — 그 사람은 아이디 + 초기 비밀번호 `dhqmsfna` 로
> 로그인한 뒤 안내에 따라 본인 비밀번호를 설정합니다(설정 전에는 예약 조회/처리 불가).
> - 비밀번호는 시트에 원문이 아니라 **암호화(해시, `hmac:...`)** 로 저장됩니다.
>   비밀번호 칸에 직접 적어도 서버가 읽는 즉시 해시로 바꿔 저장해요.
> - 확정/취소 시 **누가 처리했는지** 예약 행의 '처리자' 칸과 `처리기록` 탭에 아이디로
>   기록되고, 관리자 페이지에서는 **닉네임으로 변환**되어 표시됩니다.
> - 아이디/비밀번호/닉네임 변경: 관리자 페이지 우측 상단 **"내 정보 변경"** 메뉴.
>   아이디를 바꾸면 과거 처리자 기록의 아이디도 자동으로 함께 바뀝니다.
>   비밀번호를 잊었으면 시트에서 그 사람의 비밀번호 칸을 비우면 초기 비밀번호로 재설정 가능.
> - ⚠️ 아이디를 추가했으면 **바로 그 사람에게 초기 비밀번호로 로그인해 설정하게** 하세요.
>   (비어 있는 동안은 초기 비밀번호를 아는 누구나 그 계정을 선점할 수 있습니다)
> - ⚠️ 해시는 "읽기 보호"입니다 — 시트 **편집** 권한이 있는 사람은 자기 계정을 추가할 수 있으니
>   시트 편집 권한은 운영자만 가져야 합니다.
> (`관리자` 탭이 비어 있으면 위의 `ADMIN_TOKEN`/`ADMIN_TOKENS` 속성으로 로그인합니다)

> 이 값들은 구글 서버에만 저장되고 브라우저로 전송되지 않아 안전합니다.
> `CALENDAR_ID` 를 비워두면 캘린더 연동 없이 **시트에만** 저장됩니다.

## 4. 웹앱 배포
1. 오른쪽 위 **배포 → 새 배포**
2. 유형 선택(톱니) → **웹 앱**
3. 설정:
   - 설명: `오븐룸 예약 API`
   - **다음 사용자로 실행: 나(본인)**
   - **액세스 권한이 있는 사용자: 모든 사용자**  ← 고객이 로그인 없이 예약 가능
4. **배포** → 처음이면 권한 승인(캘린더/시트 접근) 팝업 허용
5. 발급된 **웹 앱 URL**(`.../exec`) 복사

## 5. 프론트엔드 연결
1. `js/config.js` 의 `API_URL` 에 4번 웹 앱 URL(`.../exec`) 붙여넣기
2. `js/config.js` 의 `CALENDAR_EMBED_URL` 에 1-4단계에서 복사한 삽입용 URL 붙여넣기
   (`https://calendar.google.com/calendar/embed?src=...`) — 넣으면 '예약 현황'에 캘린더가 표시됩니다.
3. `OPEN_HOUR / CLOSE_HOUR / SLOT_MINUTES` 등 운영값 확인
   (요금은 시트의 `요금표` 탭에서 관리 — `config.js` 의 `PRICING` 은 통신 실패 시 예비값)
4. 커밋 후 푸시 → **GitHub Pages** 로 자동 배포 (현재 이 레포가 그렇게 설정돼 있습니다)
   - 로컬 테스트: 프로젝트 폴더에서 `python -m http.server` 후 `localhost:8000`

## 6. 사용
- 고객: `index.html`
- 관리자: `admin.html` → 3-1의 `ADMIN_TOKEN` 입력

## 7. 개인정보 자동 파기 트리거 (1회 설정)
개인정보 처리방침(`privacy.html`)에 따라 이용일로부터 **5년** 지난 예약의 이름·연락처를 자동 삭제합니다.
1. Apps Script 에디터 상단의 함수 선택 드롭다운에서 **`setupPurgeTrigger`** 선택 → **실행** (딱 한 번)
2. 처음이면 권한 승인 팝업 허용
3. 이후 매일 새벽 4시경 `purgeOldPersonalData` 가 자동 실행됩니다.
   (왼쪽 시계 아이콘(트리거) 메뉴에서 등록 여부를 확인할 수 있어요)
- 보유 기간을 바꾸려면 `Code.gs` 의 `RETENTION_YEARS` 와 `privacy.html` 의 안내 문구를 **함께** 수정하세요.

## 코드 수정 후 재배포
Code.gs 를 고치면 **배포 → 배포 관리 → 기존 배포 편집(연필) → 버전: 새 버전 → 배포**
로 갱신하세요. (URL 은 그대로 유지됩니다.)

## 자주 겪는 문제
- **응답을 못 읽음 / CORS 오류**: 배포 액세스가 "모든 사용자"인지, URL 이 `/exec`로 끝나는지 확인.
- **권한 오류**: 4번 배포 시 권한 승인을 건너뛰었을 수 있음. 재배포하며 승인.
- **시간이 밀림**: `appsscript.json` 의 `timeZone` 이 `Asia/Seoul` 인지 확인.
- **연락처 앞 0이 사라짐** (01012345678 → 1012345678): 시트가 숫자로 자동 변환한 것.
  에디터에서 **`fixSheetFormats`** 함수를 한 번 실행하면 연락처 열이 텍스트로 고정되고
  이미 잘못 저장된 번호도 복구됩니다.
